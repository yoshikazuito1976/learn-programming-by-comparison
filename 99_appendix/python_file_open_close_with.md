# Python: ファイルを開く・閉じる・with文

この補足では、Pythonでファイルを扱うときの基本的な考え方を確認します。

## ファイル操作の基本

ファイルを読み書きするときは、基本的に次の流れになります。

1. ファイルを開く
2. ファイルを読む、または書く
3. ファイルを閉じる

たとえば、ファイルを読む処理は次のように書けます。

    file = open("sample.txt", "r", encoding="utf-8")
    text = file.read()
    file.close()

このコードでは、最初に `open()` でファイルを開いています。

    file = open("sample.txt", "r", encoding="utf-8")

次に、`read()` でファイルの中身を読み込んでいます。

    text = file.read()

最後に、`close()` でファイルを閉じています。

    file.close()

## なぜファイルを閉じる必要があるのか

ファイルを開くと、Pythonのプログラムはそのファイルを使うための準備をします。

ファイルを開いたままにしておくと、次のような問題が起こることがあります。

- 他のプログラムがそのファイルを使いにくくなる
- 書き込んだ内容がすぐに保存されないことがある
- 開いたファイルが増えすぎると、プログラムやOSに負担がかかる

そのため、ファイルを使い終わったら閉じる必要があります。

## close() を書き忘れる問題

次のコードでは、ファイルを開いていますが、閉じていません。

    file = open("sample.txt", "r", encoding="utf-8")
    text = file.read()

短いプログラムでは問題が見えにくいこともあります。

しかし、ファイルをたくさん扱うプログラムや、長く動き続けるプログラムでは、閉じ忘れが問題になることがあります。

## with文を使う

Pythonでは、ファイルを扱うときに `with` 文を使うことがよくあります。

    with open("sample.txt", "r", encoding="utf-8") as file:
        text = file.read()

    print(text)

この書き方では、`with` の中でファイルを開きます。

    with open("sample.txt", "r", encoding="utf-8") as file:

そして、インデントされた範囲の中でファイルを使います。

    with open("sample.txt", "r", encoding="utf-8") as file:
        text = file.read()

このインデントされた範囲が終わると、Pythonが自動的にファイルを閉じてくれます。

つまり、次の処理をPythonが安全に行ってくれます。

1. ファイルを開く
2. ファイルを使う
3. ファイルを閉じる

## with文を使うメリット

`with` 文を使うと、次のようなメリットがあります。

- `close()` の書き忘れを防げる
- エラーが起きてもファイルを閉じやすい
- ファイルを使う範囲がわかりやすい
- コードが安全で読みやすくなる

そのため、Pythonでファイルを扱うときは、基本的に `with open(...) as file:` を使うのがよいです。

## 比較

### close() を自分で書く場合

    file = open("sample.txt", "r", encoding="utf-8")
    text = file.read()
    file.close()

この書き方では、`close()` を忘れないようにする必要があります。

### with文を使う場合

    with open("sample.txt", "r", encoding="utf-8") as file:
        text = file.read()

この書き方では、ファイルを使い終わると自動的に閉じられます。

## イメージ

ファイルを開くことは、本を机の上に広げるようなものです。

読み終わったら、本を閉じて本棚に戻す必要があります。

`with` 文は、読み終わったあとに自動で本を閉じてくれる仕組みだと考えるとよいです。

## この補足のポイント

- ファイル操作は「開く → 読む/書く → 閉じる」の流れで考える
- `open()` はファイルを開く処理である
- `close()` はファイルを閉じる処理である
- ファイルを閉じ忘れると問題が起こることがある
- `with` 文を使うと、ファイルを自動的に閉じてくれる
- Pythonでは、ファイル操作には基本的に `with open(...) as file:` を使う
