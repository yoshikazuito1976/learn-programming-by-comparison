# Shell: 環境変数

この補足では、Shellの環境変数について確認します。

## 環境変数とは

環境変数とは、Shellやプログラムが動くときに参照する設定情報です。

たとえば、次のような情報が環境変数として用意されています。

- 現在のユーザー名
- ホームディレクトリの場所
- 使用しているShell
- コマンドを探す場所

代表的な環境変数には、次のようなものがあります。

    USER
    HOME
    SHELL
    PATH

## 環境変数を表示する

環境変数も、通常の変数と同じように `$` を付けて中身を表示できます。

    echo "$USER"
    echo "$HOME"
    echo "$SHELL"
    echo "$PATH"

たとえば、`HOME` にはホームディレクトリのパスが入っています。

    echo "$HOME"

## 環境変数の一覧を見る

環境変数の一覧は、次のコマンドで確認できます。

    env

または、

    printenv

特定の環境変数だけを見る場合は、次のようにします。

    printenv HOME
    printenv PATH

## PATHとは

`PATH` は、Shellがコマンドを探す場所の一覧です。

たとえば、次のように入力したとします。

    ls

このときShellは、`PATH` に登録されているディレクトリの中から `ls` というコマンドを探します。

`ls` の場所は次のように確認できます。

    which ls

たとえば、次のように表示されることがあります。

    /usr/bin/ls

つまり、`ls` は `/usr/bin` の中にあるコマンドです。

## 通常の変数と環境変数の違い

Shellでは、次のように変数を作ることができます。

    message="Hello"

この変数は、基本的には現在のShellの中で使う変数です。

    echo "$message"

一方、`export` を使うと、環境変数として扱うことができます。

    export message="Hello"

環境変数にすると、そのShellから起動した別のプログラムにも値を渡せるようになります。

## export の例

次のように環境変数を作ります。

    export MY_NAME="ITO"

表示してみます。

    echo "$MY_NAME"

環境変数として登録されているか確認します。

    printenv MY_NAME

## 変数を消す

変数を消すには `unset` を使います。

    unset MY_NAME

確認します。

    echo "$MY_NAME"

何も表示されなければ、変数は削除されています。

## この補足のポイント

- 環境変数は、Shellやプログラムが参照する設定情報である
- `HOME`、`USER`、`SHELL`、`PATH` などが代表例である
- 環境変数も、使うときは `$HOME` のように `$` を付ける
- 環境変数の一覧は `env` や `printenv` で確認できる
- `PATH` は、Shellがコマンドを探す場所の一覧である
- `export` を使うと、変数を環境変数として扱える
