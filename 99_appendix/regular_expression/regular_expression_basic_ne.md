# रेगुलर एक्सप्रेशनको आधारभूत कुरा

## रेगुलर एक्सप्रेशन भनेको के हो?

रेगुलर एक्सप्रेशन भनेको पाठमा ढाँचा जनाउने एउटा तरिका हो।

Linux मा यो मुख्य रूपमा `grep` जस्ता कमान्डहरूसँग प्रयोग गरिन्छ।

उदाहरणका लागि, तलको कमान्डले `students.txt` भित्र `a` भएको लाइन खोज्छ।

```bash
grep 'a' students.txt
```

## उदाहरण फाइल

पहिले अभ्यासका लागि एउटा फाइल बनाऔं।

## `grep` को आधार

### `a` भएको लाइन देखाउने

```bash
grep 'a' students.txt
```

नतिजाको उदाहरण:

```text
sato
tanaka
kato
saito
sasaki
yamada
yamamoto
alice
ai
```

`AI` ठूलो अक्षरमा भएकाले `grep 'a'` ले देखाउँदैन।

## धेरै प्रयोग हुने रेगुलर एक्सप्रेशन

| ढाँचा | अर्थ | उदाहरण |
|---|---|---|
| `a` | `a` समावेश भएको | `grep 'a' students.txt` |
| `^s` | `s` बाट सुरु हुने | `grep '^s' students.txt` |
| `o$` | `o` मा अन्त्य हुने | `grep 'o$' students.txt` |
| `.` | कुनै पनि 1 अक्षर | `grep 'a.o' students.txt` |
| `[abc]` | `a`, `b`, वा `c` मध्ये कुनै 1 अक्षर | `grep '[abc]' students.txt` |
| `[0-9]` | 1 अंक | `grep '[0-9]' students.txt` |
| `[a-z]` | 1 सानो अंग्रेजी अक्षर | `grep '[a-z]' students.txt` |
| `[A-Z]` | 1 ठूलो अंग्रेजी अक्षर | `grep '[A-Z]' students.txt` |
| `[^0-9]` | अंक बाहेकको 1 अक्षर | `grep '[^0-9]' students.txt` |
| `*` | अघिल्लो अक्षर 0 पटक वा धेरै पटक आउन सक्छ | `grep 'a*' students.txt` |
| `+` | अघिल्लो अक्षर 1 पटक वा धेरै पटक आउँछ | `grep -E 'a+' students.txt` |
| `?` | अघिल्लो अक्षर 0 पटक वा 1 पटक आउँछ | `grep -E 'a?' students.txt` |
| `a\|b` | `a` वा `b` | `grep -E 'a|b' students.txt` |

## `grep` र `grep -E` को फरक

साधारण `grep` मा `+`, `?`, `|` जस्ता चिन्हहरू सिधै प्रयोग गर्न कम सहज हुन्छ, त्यसैले `-E` थप्दा विस्तारित रेगुलर एक्सप्रेशन लेख्न सजिलो हुन्छ।

```bash
grep -E 'student[0-9]+' students.txt
```

यसले `student` पछाडि एक वा एकभन्दा बढी अंक भएका लाइनहरू खोज्छ।

## आधारभूत उदाहरणहरू

### `s` बाट सुरु हुने लाइन

```bash
grep '^s' students.txt
```

### `o` मा अन्त्य हुने लाइन

```bash
grep 'o$' students.txt
```

### अंक भएको लाइन

```bash
grep '[0-9]' students.txt
```

### `student` पछि अंक भएको लाइन

```bash
grep -E '^student[0-9]+$' students.txt
```

### साना अक्षर मात्र भएका लाइन

```bash
grep -E '^[a-z]+$' students.txt
```

### ठूला अक्षर मात्र भएका लाइन

```bash
grep -E '^[A-Z]+$' students.txt
```

### `_` वा `-` भएको लाइन

```bash
grep -E '[_-]' students.txt
```

### ठ्याक्कै 4 अक्षर भएका लाइन मात्र देखाउने

```bash
grep -E '^....$' students.txt
```

`.` ले कुनै पनि 1 अक्षर जनाउँछ।

त्यसैले `^....$` को अर्थ "सुरुदेखि अन्त्यसम्म ठीक 4 अक्षर" हो।

### ठूलो र सानो अक्षर छुट्याउन नपर्ने

```bash
grep -i 'ai' students.txt
```

यस अवस्थामा `AI` र `ai` दुवै देखिन्छन्।

### `user` नभएका लाइन देखाउने

```bash
grep -v 'user' students.txt
```

`-v` भनेको सर्तसँग नमेल्ने लाइनहरू देखाउने विकल्प हो।

## ध्यान दिनुपर्ने कुरा

रेगुलर एक्सप्रेशनमा केही चिन्हहरूको विशेष अर्थ हुन्छ।

उदाहरणका लागि, `.` यहाँ साधारण पूर्णविराम होइन। यसको अर्थ "कुनै पनि 1 अक्षर" हो।

यदि तपाईंले वास्तविक `.` खोज्न चाहनुहुन्छ भने `\.` लेख्नुहोस्।

```bash
grep '\.' file.txt
```

## सुरुमा सम्झनुपर्ने कुरा

सुरुमा यी पाँचवटा कुरा सम्झे पुग्छ।

| ढाँचा | अर्थ |
|---|---|
| `^` | लाइनको सुरु |
| `$` | लाइनको अन्त्य |
| `.` | कुनै पनि 1 अक्षर |
| `[0-9]` | अंक |
| `[a-z]` | सानो अंग्रेजी अक्षर |

## अभ्यास प्रश्नहरू

तलका सर्तहरू मिल्ने लाइन `grep` प्रयोग गरेर देखाउनुहोस्।

1. `a` भएको लाइन
2. `s` बाट सुरु हुने लाइन
3. `o` मा अन्त्य हुने लाइन
4. अंक भएको लाइन
5. `student` बाट सुरु भएर पछि अंक भएका लाइन
6. साना अक्षर मात्र भएका लाइन
7. ठूला अक्षर मात्र भएका लाइन
8. `_` वा `-` भएका लाइन
9. 4 अक्षर भएका लाइन
10. `ai` भएको लाइन, ठूलो र सानो अक्षर नमानेर
11. `user` नभएको लाइन
12. `python` पछि अंक भएको लाइन

## नमुना उत्तरहरू

```bash
grep 'a' students.txt
grep '^s' students.txt
grep 'o$' students.txt
grep '[0-9]' students.txt
grep -E '^student[0-9]+$' students.txt
grep -E '^[a-z]+$' students.txt
grep -E '^[A-Z]+$' students.txt
grep -E '[_-]' students.txt
grep -E '^....$' students.txt
grep -i 'ai' students.txt
grep -v 'user' students.txt
grep -E '^python[0-9]+$' students.txt
```

## `man` हेर्ने

थप विवरण हेर्न तलको कमान्ड प्रयोग गर्न सकिन्छ।

```bash
man grep
```

सुरुमा सबै अंग्रेजी नबुझे पनि ठिक छ।

बिस्तारै सिक्दै जानुहोस्, र चिनेका शब्द र चिन्हहरू बढाउँदै जानुहोस्।

## सारांश

रेगुलर एक्सप्रेशन एकैचोटि सबै कण्ठ गर्नुपर्ने कुरा होइन।

आवश्यक पर्दा बिस्तारै प्रयोग गर्दै जानुहोस्।

सुरुमा `grep` प्रयोग गरेर स्ट्रिङ खोज्ने कामबाट सुरु गर्नुहोस्।
