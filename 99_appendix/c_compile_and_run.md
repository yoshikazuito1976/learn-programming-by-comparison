# コンパイルと実行

C言語のプログラムは、PythonやShell Scriptのように、そのまま実行することはできません。
まず、コンパイラを使って、実行できるファイルに変換する必要があります。

この変換作業を **コンパイル** といいます。

---

### コンパイルの基本形

この教材では、次の形でコンパイルします。

```bash
gcc -Wall -Wextra ファイル名.c -o ファイル名.out
```

たとえば、`for_array.c` をコンパイルする場合は、次のように入力します。

```bash
gcc -Wall -Wextra for_array.c -o for_array.out
```

コンパイルに成功すると、`for_array.out` という実行ファイルが作成されます。

---

### 実行の基本形

作成された実行ファイルは、次のように実行します。

```bash
./ファイル名.out
```

たとえば、`for_array.out` を実行する場合は、次のように入力します。

```bash
./for_array.out
```

---

### コンパイルから実行までの流れ

C言語では、次のような流れでプログラムを動かします。

```text
C言語のソースコードを書く
        ↓
gccでコンパイルする
        ↓
実行ファイルが作られる
        ↓
実行ファイルを実行する
```

例：

```bash
gcc -Wall -Wextra for_array.c -o for_array.out
./for_array.out
```

---

### コマンドの意味

```bash
gcc -Wall -Wextra for_array.c -o for_array.out
```

このコマンドの意味は、次の通りです。

| 部分              | 意味              |
| --------------- | --------------- |
| `gcc`           | C言語のコンパイラ       |
| `-Wall`         | 基本的な警告を多めに表示する  |
| `-Wextra`       | 追加の警告も表示する      |
| `for_array.c`   | コンパイルするC言語のファイル |
| `-o`            | 出力ファイル名を指定する    |
| `for_array.out` | 作成される実行ファイル     |

---

### なぜ `-Wall -Wextra` をつけるのか

C言語では、コンパイルできても、必ず正しいプログラムとは限りません。

たとえば、次のような問題があっても、コンパイル自体は通ってしまうことがあります。

- 変数の使い方が不自然
- 危険な書き方をしている
- 意図しない型の扱いをしている
- プログラムとしては動くが、間違いにつながりやすい書き方をしている

そこで、この教材では `-Wall -Wextra` をつけて、警告を見やすくします。

```bash
gcc -Wall -Wextra ファイル名.c -o ファイル名.out
```

警告はエラーではないため、実行ファイルが作られることもあります。
しかし、警告が出た場合は、内容を確認する習慣をつけましょう。

---

### なぜ `.out` で終わる名前にするのか

この教材では、実行ファイル名を `.out` で終わる名前にします。

例：

```bash
gcc -Wall -Wextra for_array.c -o for_array.out
```

これは、`.gitignore` で実行ファイルを管理対象から外しやすくするためです。

たとえば、`.gitignore` に次のように書いておくと、

```gitignore
*.out
```

`.out` で終わるファイルはGitの管理対象から外れます。

そのため、次のような実行ファイルはGitに追加されません。

```text
for_array.out
while_count.out
loop_until_ok.out
sizeof_items.out
```

ソースコードである `.c` ファイルはGitで管理し、コンパイルして作られた `.out` ファイルはGitで管理しない、という考え方です。

---

### コンパイルエラーと警告の違い

コンパイルしたときに、エラーや警告が表示されることがあります。

#### コンパイルエラー

コンパイルエラーは、文法や型などに問題があり、実行ファイルを作れない状態です。

例：

```text
error:
```

この場合、プログラムは実行できません。
エラーの内容を確認して、ソースコードを修正する必要があります。

#### 警告

警告は、実行ファイルは作れるかもしれないが、危険な書き方や不自然な書き方があることを知らせるものです。

例：

```text
warning:
```

警告が出ても、実行ファイルが作られることがあります。
しかし、警告を無視してよいわけではありません。

C言語では、警告の中に大事なヒントが含まれていることがあります。

---

### コンパイルできても正しいとは限らない

C言語では、コンパイルできたからといって、プログラムが正しく動くとは限りません。

たとえば、次のような間違いは、コンパイルエラーにならないことがあります。

- `while` 文の中で変数を増やし忘れて、無限ループになる
- 配列の範囲外にアクセスする
- `length` の値を間違える
- 意図しないデータを読んでしまう

つまり、C言語では次の2つを分けて考える必要があります。

```text
コンパイルできること
正しく動くこと
```

この2つは同じではありません。

---

### 無限ループになった場合

プログラムを実行したあと、処理が終わらなくなることがあります。
そのような状態を **無限ループ** といいます。

たとえば、`while` 文の中で変数を増やし忘れると、条件がずっと変わらず、ループが終わらないことがあります。

実行中のプログラムを止めたい場合は、ターミナルで次のキーを押します。

```text
Ctrl + C
```

これは、実行中のプログラムを中断する操作です。

ただし、毎回 `Ctrl + C` に頼るのではなく、コードを書くときに次の点を確認しましょう。

- `while` の条件は、いつか成り立たなくなるか
- ループの中で変数は変化しているか
- `for` 文の開始値・条件・増加の書き方は正しいか
- 配列の範囲を超えてアクセスしていないか

```bash
./for_array.out
```

```bash
./loop_until_ok.out
```


---

### まとめ

C言語のプログラムを実行するときは、次の流れを意識しましょう。

```bash
gcc -Wall -Wextra ファイル名.c -o ファイル名.out
./ファイル名.out
```

大切なポイントは次の通りです。

- C言語は、まずコンパイルしてから実行する
- この教材では `-Wall -Wextra` をつけて警告を確認する
- 実行ファイルは `.out` で終わる名前にする
- `.out` ファイルはGitで管理しない
- コンパイルできても、正しく動くとは限らない
- 無限ループになったら `Ctrl + C` で止める

C言語では、プログラムを書くことだけでなく、コンパイルして、警告を読み、実行結果を確認することも大切な学習です。
