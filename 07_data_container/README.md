この章では、複数のデータをまとめて扱う方法を学びます。

これまでの章では、次の内容を学びました。

- 画面に文字を表示する
- 変数を使う
- 条件分岐を使う
- 繰り返し処理を使う
- 入力を受け取る
- 関数を使う

しかし、実際のプログラムでは、1つの値だけではなく、複数の値をまとめて扱う場面が多くあります。

たとえば、次のようなデータです。

- 学生の点数一覧
- 商品名の一覧
- ユーザー情報
- 設定値のまとまり
- 重複を取り除いたデータ
- 順番に処理するデータ

このような複数のデータをまとめて扱うための仕組みを、ここでは「データコンテナ」と呼びます。

## データコンテナとは

データコンテナとは、複数のデータを入れておくための入れ物です。

たとえば、Python では次のようなデータコンテナがあります。

| 種類 | 説明 |
|---|---|
| `list` | 複数の値を順番に並べる |
| `tuple` | 変更しない複数の値をまとめる |
| `dict` | キーと値の組み合わせで管理する |
| `set` | 重複しない値の集まりを扱う |

言語によって名前や書き方は違いますが、考え方は似ています。

## Python での例

Python では、リストを使うと複数の値をまとめて扱えます。

```python
scores = [80, 65, 90, 72]

for score in scores:
    print(score)
```

このように、1つの変数の中に複数の値を入れることができます。

### リストの例

```python
fruits = ["apple", "banana", "orange"]

print(fruits[0])
print(fruits[1])
print(fruits[2])
```

実行結果:

```text
apple
banana
orange
```

リストでは、先頭の番号は 0 から始まります。

### 辞書の例

```python
student = {
    "name": "Sato",
    "score": 80,
    "passed": True,
}

print(student["name"])
print(student["score"])
```

辞書では、キーを使って値を取り出します。

### 集合の例

```python
names = {"sato", "suzuki", "sato", "tanaka"}

print(names)
```

集合では、重複した値は 1 つにまとめられます。

## スタックの考え方

リストを使うと、スタックのような動きを表すこともできます。

スタックとは、あとから入れたものを先に取り出す仕組みです。

これを LIFO と呼びます。

LIFO は、Last In, First Out の略です。

```python
stack = []

stack.append(10)
stack.append(20)
stack.append(30)

print(stack.pop())
print(stack.pop())
print(stack.pop())
```

実行結果:

```text
30
20
10
```

最後に入れた 30 が、最初に取り出されています。

## 逆ポーランド記法との関係

スタックの考え方は、逆ポーランド記法の計算でも使われます。

たとえば、次の式を考えます。

```text
3 4 +
```

これは、通常の書き方では次の意味になります。

```text
3 + 4
```

逆ポーランド記法では、数字を順番にスタックへ入れていきます。

- 3 を入れる
- 4 を入れる
- `+` が来たら、直前の 2 つを取り出して足す
- 結果の 7 をスタックに戻す

このように、データコンテナは単にデータを入れるだけではなく、処理の考え方にもつながります。
