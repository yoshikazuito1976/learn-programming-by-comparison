# Shell: Variables

この章では、Shell における変数の基本を確認します。

Shell の変数は、他のプログラミング言語と少し書き方が異なります。

特に大切なのは、次の3点です。

- 変数に値を入れるときは `$` を付けない
- 変数の中身を使うときは `$` を付ける
- 変数を使うときは、基本的に `"$変数名"` の形にする

## 基本形

```sh
name="ITO"
echo "$name"
```

実行結果:

```text
ITO
```

## 代入するときは `$` を付けない

Shell では、変数に値を入れるときは `$` を付けません。

```sh
name="ITO"
```

これは、`name` という変数に `ITO` という文字列を入れています。

次のようには書きません。

```sh
$name="ITO"
```

これは変数への代入ではありません。

## 変数の中身を使うときは `$` を付ける

変数の中身を表示するときは `$` を付けます。

```sh
echo "$name"
```

この `$name` は、変数 `name` の中身に置き換えられます。
つまり Shell から見ると、次のような意味になります。

```sh
echo "ITO"
```

## `$name` だけを実行するとどうなるか

次のように入力すると、エラーになることがあります。

```sh
$name
```

これは、`$name` が `ITO` に置き換えられたあと、Shell が `ITO` というコマンドを実行しようとするためです。

```text
command not found
```

変数の中身を表示したい場合は `echo` を使います。

```sh
echo "$name"
```

## シングルクォートとダブルクォート

Shell では、クォートの種類によって動きが変わります。

```sh
name="ITO"
echo "$name"
echo '$name'
```

実行結果:

```text
ITO
$name
```

- ダブルクォート `"..."` では、変数が展開される
- シングルクォート `'...'` では、変数は展開されない

## 比較

```sh
name="ITO"

echo $name
echo "$name"
echo '$name'
```

実行結果:

```text
ITO
ITO
$name
```

基本的には、変数を使うときは `"$name"` のようにダブルクォートを付けるのが安全です。

## スペースを含む値

次のように、値にスペースが含まれる場合があります。

```sh
message="Hello Shell"
```

この変数を使うときは、ダブルクォートを付けます。

```sh
echo "$message"
```

ダブルクォートを付けることで、`Hello Shell` を1つのまとまりとして扱えます。

## サンプルコード

```sh
name="ITO"
language="Shell"

echo "名前: $name"
echo "言語: $language"
echo "$name is learning $language."
```

## 補足: 環境変数

Shell では、自分で作る変数のほかに環境変数があります。
環境変数は、Shell やプログラムが動くときに参照する設定情報のようなものです。

代表的な環境変数:

```text
PATH
HOME
USER
SHELL
```

確認例:

```sh
echo "$USER"
echo "$HOME"
echo "$PATH"
```

一覧表示:

```sh
env
printenv
```

通常の変数との違いを簡単に言うと、環境変数は他のプログラムにも渡される変数です。

```sh
message="Hello"          # 現在の Shell で使う変数
export message="Hello"   # 子プロセスにも渡せる環境変数
```

## クォートの種類

シェルでは、文字列を書くときにクォートを使います。
クォートの種類によって意味が変わります。

| 書き方 | 名前 | 意味 |
|---|---|---|
| `"..."` | ダブルクォート | 文字列として扱う。ただし変数は展開される |
| `'...'` | シングルクォート | 文字列として扱う。変数もそのまま表示される |
| `` `...` `` | バッククォート | コマンドを実行して、その結果に置き換える古い書き方 |
| `$(...)` | コマンド置換 | コマンドを実行して、その結果に置き換える現在の書き方 |

### ダブルクォート

```sh
name="ITO"
echo "$name"
```

実行結果:

```text
ITO
```

### シングルクォート

```sh
name="ITO"
echo '$name'
```

実行結果:

```text
$name
```

### コマンド置換

```sh
echo "Today: $(date +%F)"
```

バッククォートでも同じことはできますが、`$(...)` のほうが読みやすく、入れ子にも対応しやすいため推奨です。

## この章のポイント

- Shell では、変数に値を入れるとき `$` は付けない
- 変数の中身を使うとき `$` を付ける
- 変数は `"$name"` のようにダブルクォート付きで使うのが安全
- シングルクォートでは変数は展開されない
- `$name` だけを書くと、変数の中身をコマンドとして実行しようとすることがある
