# Shell: Variables

この章では、Shellにおける変数の基本を確認します。

Shellの変数は、他のプログラミング言語と少し書き方が異なります。

特に大切なのは、次の点です。

- 変数に値を入れるときは `$` を付けない
- 変数の中身を使うときは `$` を付ける
- 変数を使うときは、基本的に `"$変数名"` の形にする

## 基本形

    name="ITO"

    echo "$name"

実行すると、次のように表示されます。

    ITO

## 代入するときは `$` を付けない

Shellでは、変数に値を入れるときは `$` を付けません。

    name="ITO"

これは、`name` という変数に `ITO` という文字列を入れています。

次のように書いてはいけません。

    $name="ITO"

これは変数への代入ではありません。

## 変数の中身を使うときは `$` を付ける

変数の中身を表示するときは `$` を付けます。

    echo "$name"

この `$name` は、変数 `name` の中身に置き換えられます。

つまり、Shellから見ると次のような意味になります。

    echo "ITO"

## `$name` だけを実行するとどうなるか

次のように入力すると、エラーになることがあります。

    $name

これは、`$name` が `ITO` に置き換えられたあと、Shellが `ITO` というコマンドを実行しようとするためです。

    ITO

しかし、`ITO` というコマンドは存在しないため、次のようなエラーになります。

    command not found

変数の中身を表示したい場合は、`echo` を使います。

    echo "$name"

## シングルクォートとダブルクォート

Shellでは、クォートの種類によって動きが変わります。

    echo "$name"

ダブルクォートでは、変数が展開されます。

    ITO

一方、シングルクォートでは、変数は展開されません。

    echo '$name'

実行すると、次のようにそのまま表示されます。

    $name

## 比較

    name="ITO"

    echo $name
    echo "$name"
    echo '$name'

結果は次のようになります。

    ITO
    ITO
    $name

基本的には、変数を使うときは `"$name"` のようにダブルクォートを付けるのが安全です。

## スペースを含む値

次のように、値にスペースが含まれる場合があります。

    message="Hello Shell"

この変数を使うときは、ダブルクォートを付けます。

    echo "$message"

ダブルクォートを付けることで、`Hello Shell` を1つのまとまりとして扱うことができます。

## サンプルコード

    name="ITO"
    language="Shell"

    echo "名前: $name"
    echo "言語: $language"
    echo "$name is learning $language."

## この章のポイント

- Shellでは、変数に値を入れるとき `$` は付けない
- 変数の中身を使うとき `$` を付ける
- 変数は `"$name"` のようにダブルクォート付きで使うのが安全
- シングルクォートでは変数は展開されない
- `$name` だけを書くと、変数の中身をコマンドとして実行しようとすることがある

## 補足：環境変数

Shellでは、自分で作る変数のほかに、環境変数というものがあります。

環境変数は、Shellやプログラムが動くときに参照する設定情報のようなものです。

代表的な環境変数には、次のようなものがあります。

    PATH
    HOME
    USER
    SHELL

たとえば、現在のユーザー名は次のように確認できます。

    echo "$USER"

ホームディレクトリは次のように確認できます。

    echo "$HOME"

コマンドを探す場所は `PATH` に入っています。

    echo "$PATH"

環境変数の一覧を見るには、次のコマンドを使います。

    env

または、

    printenv

通常の変数との違いを簡単に言うと、環境変数は他のプログラムにも渡される変数です。

たとえば、次のように変数を作ります。

    message="Hello"

このままだと、基本的には現在のShellの中だけで使う変数です。

一方、`export` を使うと、子プロセスにも渡せる環境変数になります。

    export message="Hello"

ただし、環境変数は少し発展的な内容です。

この章では、まず次の基本を押さえましょう。

- 変数に値を入れるときは `$` を付けない
- 変数の中身を使うときは `$` を付ける
- 環境変数も、使うときは `"$HOME"` のように `$` を付ける
